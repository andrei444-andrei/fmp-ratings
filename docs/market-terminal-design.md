# Рыночный терминал — проект раздела (design doc)

> Главная страница нового модуля. Аналог обзорных экранов Bloomberg (GMM/WEI) и
> Koyfin «Today's Markets», адаптированный под наш стек (Next.js 15 + Turso +
> кэш-первый `getPrices`) и светлую дизайн-систему «светлый финтех» (`--fk-*`).
> Статус: **проект к согласованию** (реализация не начата).
>
> **Решено:** маршрут раздела — `/terminal`, подпись в меню «Рыночный терминал».
> Новостной слой стартует **сразу с FMP/EODHD news** (рыночные аномалии — как
> сигнал ранжирования важности, а не единственный источник).

---

## 1. Обещание продукта

При заходе пользователь за **3 секунды** получает ответ на вопрос
**«что изменилось сейчас в мире и на рынках»** — не меню разделов, а готовую
картину рынка. Три уровня ответа сверху вниз:

1. **Режим рынка** — risk-on / risk-off одним взглядом (стержень).
2. **Где движение** — лидеры/аутсайдеры/аномалии дня + сетки доходностей.
3. **Почему** — лента значимых новостей, привязанных к блокам вселенной.

Приоритеты: **функционально, быстро (первый экран < 1.5 с), красиво и
интуитивно**, ориентация на качество решений управляющего.

Главная — основная, остальные экраны модуля (`/news`, `/admin/terminal`,
будущие трейды) — вспомогательные.

## 2. Архитектурные инварианты (что фиксируем до первой строки кода)

Эти пункты — результат критик-прохода: места, где наивный дизайн расходится с
реальным кодом репозитория. Закрываем их **в проекте**, чтобы не переписывать
потом.

1. **FX-инвариант вселенной.** `eodhdEod` отдаёт `adjusted_close` в **нативной
   валюте биржи**. Поэтому в блоке «Страны» используем **только US-листинги ETF**
   (EWG/EWJ/INDA/EWZ котируются в USD на NYSE Arca) — их доходность уже в единой
   базе USD (включает валютную переоценку страны, что для дашборда корректно).
   Каждому инструменту в реестре — поле `currency`; unit-тест проверяет, что у
   всех членов `countries` `currency==='USD'`. Никакого слоя FX-конвертации на
   старте не нужно.
2. **Разделение `overview` (чтение) и `refresh` (расчёт).**
   `GET /api/market/overview` **никогда не считает** — только отдаёт готовый
   snapshot (мгновенно, кэшируемо). Весь расчёт — в `POST /api/market/refresh`
   (cron/токен, `maxDuration=300`). Холодный старт без snapshot → отдаём каркас
   `blocks: loading` + триггерим refresh, UI показывает Skeleton. Это
   гарантирует <1.5 с независимо от состояния кэша на serverless.
3. **Cron обязателен, не опционален.** Заводим `vercel.json` с cron на
   `/api/market/refresh` (после закрытия US-сессии + дневной прогон для гео) и
   `/api/news/refresh`. Без крона snapshot протухает по TTL и каждый «первый за
   час» посетитель ловит холодный путь.
4. **Нормировка heatmap по столбцу/классу.** Готовый `Heatmap` нормирует по 90-му
   перцентилю **всех** ячеек — для смешанных классов (URA ±15% vs EWG ±2%) это
   «выжигает» шкалу. Добавляем проп `normalizeBy: 'all' | 'col' | 'group'`; для
   терминала дефолт — **per-column** (каждый период — своя шкала) или режим
   **z-score-окраски** (нормирует разную волатильность по построению).
5. **Дальтонизм в heatmap-режиме.** Цвет всегда дублируется нецветовым сигналом.
   В строках это `Delta` (знак + стрелка); в heatmap-ячейке добавляем
   треугольник направления/знак — чтобы дефолтный режим Секторов соответствовал
   правилу доступности.
6. **Выравнивание торговых календарей.** NYSE/XETRA/TSE имеют разные праздники.
   `avgCorr`/`breadth`/корреляции считаем по **пересечению дат** членов блока
   (inner-join по датам, forward-fill пропусков не глубже 1–2 баров). В ответе
   отдаём `asOf` по блоку + `lastBarDate` по инструменту, чтобы UI честно помечал
   «данные за вчера» для отстающих бирж.
7. **Защита demo-режима.** `syntheticSeries` даёт правдоподобную, но **выдуманную**
   картину (фейковый risk-on, breadth ~100%). Если оба ключа отсутствуют или доля
   синтетики в snapshot выше порога — глобальный баннер «демо-данные» и **Режим
   рынка скрываем** (не показываем выдуманный режим). Синтетику привязать к
   торговому календарю (пропуск сб/вс), чтобы окна совпадали с реальными.
8. **Инвалидация snapshot при изменении вселенной.** Ключ snapshot включает
   `configVersion`: `overview@{TERMINAL_EPOCH}:{configVersion}`. CRUD на
   `/admin/terminal` бампит `configVersion` → старый snapshot промахивается →
   пересборка.

Всё остальное — по конституции: самопровижининг таблиц (`CREATE TABLE IF NOT
EXISTS`), обязательный `created_at`, ошибки → `app_errors`, фичи ходят только
через кэш-слой (`getOverview`/`getNews`), graceful без ключей, epoch-инвалидация.

## 3. UX / UI

Светлая тема, наш UI-кит. Главный вывод — в **верхний-левый квадрант**.
Плотность достигается выравниванием и единообразием блоков (small multiples),
а не рамками и декором: блоки разделяются воздухом и фоном `surface-elev`.

### 3.2 Компоненты

**Готовые (переиспользуем):** `Card/CardHeader/CardTitle/CardContent` (блоки),
`Delta` (доходности со знаком+стрелкой — нецветовой сигнал), `Sparkline` (тренд

`MiniSpark` (фикс. 44×16 `Sparkline`), `RangeBar` (52w-range мини-бар),
`DetailDrawer` (на базе `Modal`-портала, правое позиционирование).

### 3.3 Цвет и числа

- Токены `--fk-up/down/warn`; **цвет всегда дублируется** знаком/стрелкой. Под
  светлую тему и дальтонизм палитра сдвинута к teal↔amber↔rose; критичные числа —
  через `*-text` токены (контраст ≥ 7:1). Около нуля — явный нейтральный тон, не
  «невидимка».
- `tabular-nums` (Inter настроен), right-align по разрядам, явный знак,
  типографский минус `−`, единицы/`%` приглушённым `text-ink-3`, 1 знак в
  сетке / 2 в Drawer. Иерархия: KPI → под-метрики → labels.

### 3.4 Взаимодействия и состояния

- Глобальный режим **Абсолют% / Z-score / Отн.сила** через `SegmentedControl` с
  мгновенным **клиентским** пересчётом отображения (числа считаются на сервере).
- Сортировка по столбцу с сохранением при смене режима; hover-тултип; drill-down
  в Drawer с сохранением метрики; pin/watch → блок «Избранное»; ⌘K-поиск.
- Загрузка: `Skeleton`, sticky-бар грузится первым (stale-while-revalidate).
  Битые данные → `—` + `Badge warn`. Ошибки → `app_errors`. Demo → `Badge`.
- Микроанимации только из имеющихся keyframes (`overlay-in`/`modal-in`/`shimmer`/
  `toast-in`), всё ≤ 300 мс.

### 3.5 Адаптив

Десктоп: `grid-cols-2`, все периоды, Drawer 440px. Планшет: 1 колонка,
сокращённые периоды. Мобайл: карточный режим вместо таблицы, Heatmap отключён,
Drawer = полноэкранный нижний `Modal`, sticky-бар с horizontal-snap; блоки ниже
сгиба — ленивая подгрузка (контроль веса DOM/payload).

## 4. Метрики и периоды — ответ на «что-то ещё надо?»

**Окна.** `1, 5, 21, 63, 126, 252` торговых дня (заменяем `7→5` = торговая
неделя). Считаем **по индексам массива торговых дней** (`closes[len-1]` vs
`closes[len-1-N]`), не по календарю. Плюс календарные **MTD/QTD/YTD** (не
дублируют скользящие окна, нужны инвесткомитету; считаются от первого бара
на/после границы периода). Интрадей — отложить (у нас EOD `adjusted_close`).

**Минимальная сетка (~7 полей на инструмент):**

| Поле | Что показывает |
|---|---|
| `ret 1D / 21D / 63D` | импульс на трёх горизонтах |
| `retYTD` | результат с начала года |
| `excess63` | **относительная сила** vs бенчмарк блока (главное, чего нет сейчас) |
| `vol21` + флаг `volRatio` | риск + ускорение волатильности (vol21/vol63 — смена режима) |
| `pct52w` + цвет `>MA200` | положение в диапазоне (самый читаемый risk-индикатор) |
| `z63` как подсветка `⚡` | аномалия: `z=(r−mean)/std`, lookback 63д, подсветка `|z|>2` |

`vol21/vol63` — annualized из дневных лог-доходностей (×√252).

**Метрики блока (в шапке, считаются на сервере по матрице членов):** breadth
(`%>MA50/MA200`, advancers/decliners, composite 0–100), `avgCorr` (средняя
попарная корреляция 63д), для малых корзин — equal-weight доходность + лучший/
худший компонент.

**Глобальный режим (виджет наверху):** `avgCorr` + vol-режим + breadth.

**Расширенный набор — только в Drawer:** полная кривая 1/5/21/63/126/252 +
MTD/QTD/YTD + annualized 126/252; RS-Ratio/RS-Momentum (RRG-квадрант);
vol21/63/ratio, MDD252, Sortino, ATR; MA50/MA200-дистанция, RSI(14); бета/корр к
SPY; z-score числом.

Все метрики — **чистые функции от `getPrices(adjusted_close)`**. Новых
провайдеров не нужно — только вычислительный/кэш-слой.

## 5. Вселенная и расширяемость

**Модель — «вселенная это данные, а не код».** Гибрид (как `signals` уже делает):
сид в коде + пользовательские override в Turso.

- **Сид-реестр** `src/lib/terminal/registry.ts` — чистые данные (паттерн
  `PresetDef`): `BlockDef { id, title, type, benchmark, members: string[], layout }`
  + `InstrumentDef { symbol, title, kind, currency, optional?, proxy?, weight? }`.
  Тикер хранится **один раз** по `symbol`; блоки ссылаются (**many-to-many**) →
  `XLU`/`URA` не дублируются, цены не тянутся дважды.
- **Override в Turso** (self-provisioning, `ensureTerminalTables()`):
  - `terminal_instruments` — инструмент один раз (symbol, kind, currency, флаги).
  - `terminal_blocks` — code/title/type/benchmark/layout/enabled.
  - `terminal_block_members` — m2m блок↔инструмент с `weight`.
  - `terminal_baskets` (сайдкар) — `composition.version`, `proxy_symbol`,
    `trade_rules`, `backtest` → **forward-compatible с будущими трейдами**.
  - Сервер сливает сид+override по `code`/`symbol` (`enabled=0` скрывает сид).
- **Корзина = блок `type='basket'`.** Доходность: (1) взвешенная по
  `members.weight` (целевая, когда появятся компоненты — NVDA/AVGO/TSM/ASML и
  т.п.), иначе (2) `proxy_symbol` — один ETF-прокси (старт). Будущий трейд-модуль
  читает `terminal_baskets` без правки главной.
- **Добавление блока (UX):** основной путь — `/admin/terminal` (CRUD +
  JSON-редактор, аналог `/admin/quantconnect`, под токеном) + инлайн «+ Блок»
  через `Modal` на самой `/terminal`. Валидация тикера переиспользует
  `GET /api/admin/eodhd-check?symbols=` (ok && count>252 && adjusted); невалидный
  не сохраняется без подтверждения.

## 6. Слой данных и производительность

**Snapshot-first (stale-while-revalidate).** Таблица `market_snapshot`
(`key, as_of, epoch, payload` (gz+base64), `refreshed_at, created_at`) отдаёт
готовый JSON мгновенно; долив цен и пересчёт — фоном через `refresh`.

- **`GET /api/market/overview`** (`force-dynamic`, только чтение): отдаёт snapshot
  + `Cache-Control: public, s-maxage=60, stale-while-revalidate=300` (CDN-уровень,
  снимает повторную распаковку gzip). Ответ: блоки → инструменты →
  метрики + спарклайн (downsample ~80 точек) + метрики блока + `regime` + `asOf`.
- **`POST /api/market/refresh`** (cron/токен, `maxDuration=300`): единственное
  место расчёта. Батч `getPrices` чанками `Promise.all` по 6 (как `signals/study`),
  окно `from = today − ~400 дней`. ~50 уникальных тикеров (m2m → тикер тянется
  один раз).
- **Graceful:** `getPrices()`; `if (s.length<5) s=syntheticSeries(sym)`. Per-symbol
  `try/catch` + `logAppError` — ошибка одного тикера не валит overview.
- **TTL** snapshot ~30–60 мин; `TERMINAL_EPOCH` для инвалидации формата метрик;
  ключ включает `configVersion` (см. §2.8).

**Имена в стиле репо:** `lib/terminal/{config,registry,metrics,block-metrics,store,
overview}.ts`; типы `InstrumentMetrics / BlockMetrics / MarketOverview /
MarketSnapshot`; константа `RET_WINDOWS=[1,5,21,63,126,252]`.

## 7. Новости с влиянием на рынок

**Решение: отдельный кэш-МОДУЛЬ + два уровня потребления** (Вариант B). Новости —
**слой-оверлей, не ячейка числовой сетки**.

- **Уровень 1 (MVP):** встроенная лента «Что движет рынком» на главной +
  точка-индикатор на блоках. Главный потребитель.
- **Уровень 2:** отдельная витрина `/news` (полная фильтруемая лента) —
  добавить в `Nav.tsx`.

**Ключевое преимущество:** «важность» новости считаем **без новостных API** —
abnormal return `|ret_1d|/vol_21d` и всплеск объёма уже есть из
`getPrices+vol21`. Текст лишь объясняет «что и почему».

**Источники (решено — стартуем сразу с новостных фидов):** FMP `stock_news` +
EODHD news+sentiment с первого дня (оба ключа уже есть), рыночный сигнал из кэша
цен (`|ret1d|/vol21` + объём) — как **сигнал важности/ранжирования**, Perplexity
Sonar через `aimlChatWithCitations` — фолбэк/дайджест и graceful без ключей.
Каскад деградации: FMP/EODHD news → рыночные аномалии → Sonar.

**Таблицы** (self-provisioning, паттерн `prices.ts`): `news_items`,
`news_clusters` (единица на главной, с `impact_score`/`blocks`/`instruments`),
`news_item_cluster` (m2m), `news_meta`. `created_at` везде, epoch.

**Pipeline:** сбор (батч тикеров + abnormal/vol) → дедуп SimHash + кластеризация
по тикерам/окну → **AI через aimlapi** (батч-классификатор JSON: ru-саммари +
direction + impact; **привязка кластера к блокам/инструментам вселенной —
киллер-фича**; Sonar-дайджест) → гибридный `impact_score` 0–100.

**Дисциплина стоимости:** AI **никогда** не зовётся синхронно на запрос — только
при обновлении кэша на новые кластеры; кэш классификации по SimHash; **потолок
N новых кластеров на цикл**; Sonar ~$0.006/вызов, TTL 10–15 мин.

**Согласование TTL.** В snapshot храним только стабильный маркер (`hasNews:
boolean + count` на блоке); конкретные кластеры — отдельным быстрым `GET /api/news`
по клику. Тогда 15-мин новости не требуют пересборки 60-мин snapshot.

**API:** `GET /api/news` (топ-кластеры), `GET /api/news/by-block`, `POST
/api/news/refresh` (cron/токен). Фасад `getNews()` — единственная точка для фич.

## 8. Топ-идеи для управляющего family office

Ранжировано по ценности для решений × реализуемости на нашем стеке (V=ценность,
E=трудозатраты, 1–10).

### P0 — стержень (дёшево, чистая математика на ценах)

1. **Глобальный риск-режим (Risk-On/Off)** — V10/E3. Композит из vol-режима +
   avgCorr 63д + breadth. Единственный виджет, дающий за 1 сек ответ «что
   изменилось в мире». Sticky-полоса над сеткой.
2. **Z-score сканер аномалий** — V9/E2. Лента «аномалии дня»: `|z|>2` и
   `vol_ratio>порог` по всей вселенной. Фокус внимания без перебора тикеров +
   триггерный слой для новостей и алертов.
3. **Дродаун-монитор и риск-панель блоков** — V9/E3. Текущий drawdown от
   252д-пика, MDD252, дни в просадке, vol21/63, %52w. Переиспользует
   `drawdowns.ts` + `computeMetrics`. Слой реализованного риска.
4. **Сравнение блоков (overlay)** — V8/E3. Выбор 2–5 блоков, overlay
   нормированных кривых total return (rebase=100), side-by-side таблица + vol +
   MDD + попарная корреляция. Прямой аллокационный инструмент.

### P1 — расширение

5. **Атрибуция и breadth компонентов блока** — V8/E4. «Движение блока — это один
   тикер-выброс или широкий тренд?». Переиспользует `breadth.ts` +
   `attribution.ts`. Качество движения, не только величина.
6. **Тепловая карта корреляций кросс-ассет** — V8/E4. Матрица 63д на готовом
   `Heatmap` + режим «дельта корреляции» (рвущиеся связи). Проверка реальной
   диверсификации.
7. **FX, ставки, форма кривой (макро-фундамент)** — V8/E4. Блок ETF-прокси: UUP,
   SHY/IEF/TLT, LQD/HYG, TIP; спред 2s10s и HY-IG, флаг инверсии. Связывает «что»
   с «почему».
8. **Утренний AI-брифинг «что важного за ночь»** — V9/E5. Сервер собирает
   топ-аномалии + лидеров/аутсайдеров + смену режима + earnings дня →
   `aimlChatWithCitations` (Sonar) пишет ru-текст с источниками. Graceful: без
   ключа только числовая сводка.
9. **Watchlist с триггерами и алертами (EOD Cron)** — V8/E5. Правила (пробой 52w
   high/low, `|z|>X`, vol_ratio, кросс MA50/MA200), EOD-прогон через Vercel Cron,
   лента «что сработало за ночь». Активный сторож портфеля.

### P2

10. **Скринеры на базе вселенной** — V7/E4. Пресеты Momentum / Mean-reversion /
    Breakout / RS-лидеры. Генератор идей под текущий режим.
