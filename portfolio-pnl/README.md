# Portfolio P&L

P&L-отчёт по личным инвестициям с разнородными активами. Показывает, как поквартально
меняется структура капитала, сводит статистику по сегментам (фондовые рынки, Private Equity,
Real Estate, Crypto, Cash) и умеет принимать **плохо структурированные данные** — через
CSV-импорт, ручной ввод и AI-парсинг произвольного текста.

## Возможности

- **Overview-дашборд** в стиле референса:
  - Net worth по кварталам (график) + дельты YoY / QoQ / с начала.
  - KPI: TWR и MWR (приближение modified-Dietz), чистые взносы, доход за период.
  - Аллокация по классам активов (донат + легенда).
  - P&L bridge — водопад вкладов сегментов между первым и последним кварталом.
  - Модули по сегментам с QoQ и нереализованным P&L.
  - Профиль ликвидности (T+0 / T+7 / T+90 / Locked).
  - Авто-алерты: тонкая кэш-подушка, просадка сегмента, устаревшая оценка, отклонение от цели.
  - Переключатель «as-of» квартала.
- **Ввод данных** (`/import`):
  - **CSV брокера** — устойчивый парсер с эвристическим сопоставлением колонок
    (IBKR/Schwab/Fidelity/T-Bank и т.п.). Нераспознанные строки автоматически уходят в AI.
  - **Ручной ввод** — форма позиции.
  - **AI «как есть»** — вставьте выписку/заметки/сообщение, модель разложит на позиции.
  - Предпросмотр с правкой класса/суммы и удалением строк перед сохранением.

## Стек

Next.js 15 · React 19 · Tailwind · Drizzle ORM + libSQL (Turso) · aimlapi.com (AI-гейт).

## Запуск локально

```bash
cp .env.example .env
# для локалки достаточно LOCAL_SQLITE_PATH=./local.db
# для AI-парсинга добавьте AIMLAPI_KEY (https://aimlapi.com/app/keys)

npm install
npm run db:migrate        # создаёт таблицы
tsx src/db/seed.ts        # (опционально) демо-данные на 6 кварталов
npm run dev               # http://localhost:3000
```

## Прод (Vercel + Turso)

1. Создайте БД: `turso db create portfolio-pnl`, возьмите URL и токен.
2. В переменных окружения задайте `TURSO_DATABASE_URL`, `TURSO_AUTH_TOKEN`, `AIMLAPI_KEY`
   (не задавайте `LOCAL_SQLITE_PATH`).
3. После деплоя примените миграции: `POST /api/db/migrate`.

## Данные

| Таблица        | Назначение                                                        |
|----------------|-------------------------------------------------------------------|
| `holdings`     | позиции по кварталам (класс, стоимость, себестоимость, ликвидность)|
| `cashflows`    | пополнения/выводы/доход — для KPI и TWR/MWR                        |
| `segment_meta` | целевая аллокация и дата последней оценки сегмента (для алертов)   |

Квартал хранится как `YYYYQn` (например `2026Q1`).

## AI-гейт

Все AI-запросы идут через OpenAI-совместимый эндпоинт aimlapi.com
(`src/lib/aimlapi.ts`). Модель задаётся через `AIMLAPI_MODEL` (по умолчанию `gpt-4o-mini`).
Парсинг неструктурированных данных — `src/lib/ai-parse.ts`.
