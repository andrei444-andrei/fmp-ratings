## Git / PR workflow
- После пуша создавать PR **сразу как обычный (не draft)** и **сразу мержить** в `main` через GitHub API, без ожидания ревью.
- Перед мержем дожидаться, что обязательные CI-проверки (если они есть) прошли успешно. Vercel Preview — необязательный, ждать его не нужно.
- Если CI упал — не мержить, разобраться и починить, потом смержить.
- Использовать squash merge по умолчанию.
- **Прод деплоится автоматически из `main`** (§9 конституции): мерж в `main` = заливка на прод (https://fmp-ratings.vercel.app), из любого чата. Задача не закрыта, пока не смержено в `main`; руками `vercel --prod` не деплоить. После мержа проверить, что прод реально обновился.
